# 正確性・バグリスク

コードが意図どおり動くか。壊れる条件が残っていないか。

## ロジック
- オフバイワン（境界の `<` / `<=`、ループの開始・終了）
- 条件分岐の網羅漏れ（else・default の欠落、想定外の値）
- 早期 return・break・continue の見落としによる処理の飛ばし
- 等価比較の取り違え（型強制、参照比較と値比較）

## 値の扱い
- null / undefined / 空文字 / 空配列の未処理
- 数値のオーバーフロー・桁落ち・ゼロ除算
- 日付・タイムゾーン・ロケール依存の処理

## 非同期・並行
- await 漏れ、Promise の握り潰し
- エラーの伝播漏れ（catch で握って無言で続行していないか）
- 競合状態・順序依存（並行更新、二重実行）
- リソースの解放漏れ（接続・ファイル・ロック）

## エラー処理
- 例外を握り潰していないか（ログも再送出もしない catch）
- 失敗時に中途半端な状態が残らないか（部分的な書き込み）
- 外部入力・外部応答を信頼していないか（バリデーション）

## 重要度の目安
- データ破壊・クラッシュ・誤った結果を生む → 🔴 MUST
- 特定条件でのみ問題、影響が限定的 → 🟡 SHOULD
- 現状は動くが将来壊れやすい → 🟢 NITS / 💬 QUESTION
