# パフォーマンス

無駄な処理・スケールしない書き方を見る。早すぎる最適化は求めない。

## クエリ・I/O
- N+1 クエリ（ループ内での DB アクセス・API 呼び出し）
- 不要な全件取得（必要なカラム・件数に絞れているか）
- インデックスの効かない検索条件
- バッチ化・キャッシュで減らせる往復

## 計算量
- ループ内の重い処理・多重ループ（データ量で破綻しないか）
- 同じ計算の繰り返し（ループ外に出せるか、メモ化できるか）
- 大きなデータの全読み込み（ストリーム・ページングの検討）

## メモリ・リソース
- 解放されないリソース・参照（リーク）
- 大量データの一括メモリ展開
- 不要なコピー・中間データの生成

## フロントエンド
- 不要な再レンダリング・再計算
- 大きなバンドルへの寄与（重い依存の安易な追加）

## 重要度の目安
- 本番のデータ量で明確に問題になる → 🔴 MUST / 🟡 SHOULD
- 影響が小さい・限定的 → 🟢 NITS
- 計測しないと判断できない → 💬 QUESTION（推測で MUST にしない）
