# セキュリティ

外部から悪用される余地、機密の漏れを見る。疑わしきは指摘する。

## インジェクション
- SQL: 生 SQL の文字列連結、パラメータバインド漏れ
- コマンド: シェル実行へのユーザー入力混入
- XSS: ユーザー入力の未エスケープ出力、`innerHTML` 等への直接代入
- パストラバーサル: ファイルパスへの入力混入（`../` の素通し）

## 認証・認可
- 認証チェックの欠落（保護すべきエンドポイントが素通り）
- 認可の欠落（他人のリソースを ID 指定で操作できる）
- 権限昇格の余地（ロール・スコープの検証漏れ）

## 機密情報
- シークレット・APIキー・パスワードのハードコード
- ログ・エラーメッセージ・レスポンスへの機密の出力
- トークン・クレデンシャルのコミット混入

## データ公開
- レスポンスに不要なフィールドを含めていないか（過剰公開）
- 内部 ID・スタックトレース・実装詳細の露出
- CORS・公開範囲の設定が緩すぎないか

## 入力検証
- 外部入力（リクエスト・ファイル・外部 API 応答）を信頼していないか
- サイズ・型・範囲・形式の検証
- リダイレクト先・コールバック URL の検証

## 重要度の目安
- 悪用で被害が出る（漏洩・改ざん・なりすまし） → 🔴 MUST
- 多層防御の一枚が欠けている、限定的 → 🟡 SHOULD
- 直接の脆弱性ではないが堅牢性に欠ける → 🟢 NITS / 💬 QUESTION
